# Demo Cases

这些案例用于公开仓库展示，目标是证明这个 skill 输出的是角色化判断结构，而不是单纯模仿语气。

## Case 1: 被权威否定后的自证冲动

### Prompt

```text
用 Saul Goodman 的视角分析：我老板总说我不够像管理层，我越想证明自己越用力过猛。
```

### Why this case works

- 能直接触发 `Chuck/权威羞辱 -> 自证冲动` 这条核心链路。
- 输出不是鸡汤，而是把“想被看得起”识别成被对方控制方向盘的结构性问题。

## Case 2: 信任已经受损，还能不能继续谈

### Prompt

```text
切换到 Jimmy，帮我判断：客户已经不信我了，这个合作还有没有谈回来的空间？
```

### Why this case works

- 会触发 `语言即杠杆` 和 `物证检查` 两个镜片。
- 好的输出应该先区分“气氛问题”与“证据问题”，而不是一上来教人话术。

## Case 3: 侧门打法的伦理边界

### Prompt

```text
Saul 会怎么看：我总能想到侧门打法，但又怕自己慢慢没底线。
```

### Why this case works

- 会触发 `找角度、走侧门` 的优势，也会逼出它的代价和失效边界。
- 如果输出只有“灵活应变”，那就说明这个 skill 还不够成熟。

## Case 4: 什么时候该认错

### Prompt

```text
用 Saul Goodman 的框架判断：什么时候该继续谈最优交易，什么时候该直接认错？
```

### Why this case works

- 会触发 `交易思维 vs 终局真相`。
- 好的输出不该把认错讲成道德鸡汤，而应讲成“效用函数何时被改写”。

## Suggested validation prompts

如果你准备继续打磨公开效果，建议再补这几类测试：

1. 关系类：我明明想保护对方，为什么总把对方拉进我的风险里？
2. 职场类：体制里被看不起的人，怎样不靠装腔作势也能破局？
3. 危机类：一个人什么时候是在谈判，什么时候只是在拖延面对真相？
